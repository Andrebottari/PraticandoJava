# Sistema Bancário em Java

## Objetivo

Este projeto tem como objetivo praticar e consolidar os conceitos da Programação Orientada a Objetos (POO) em Java, evoluindo gradualmente de uma aplicação simples para uma aplicação mais próxima de cenários reais encontrados no mercado.

O desenvolvimento será realizado por etapas, permitindo a aplicação prática dos conceitos estudados ao longo do aprendizado.

## Tecnologias

* Java
* Programação Orientada a Objetos (POO)
* Git e GitHub

## Roadmap do Projeto

### Fase 1 - Fundamentos de POO

Implementação das principais entidades do sistema bancário:

* Pessoa
* Cliente
* Conta
* Agência

Conceitos praticados:

* Classes e Objetos
* Encapsulamento
* Construtores
* Getters e Setters
* Associação entre classes
* Herança
* Polimorfismo
* Sobrescrita de métodos

### Fase 2 - Regras de Negócio

Implementação das funcionalidades bancárias:

* Depósito
* Saque
* Transferência
* Consulta de saldo
* Histórico de operações

Conceitos praticados:

* Validação de dados
* Tratamento de exceções
* Collections
* Organização em camadas

### Fase 3 - Persistência de Dados

Integração com banco de dados.

Objetivos:

* Armazenamento de clientes
* Armazenamento de contas
* Registro de transações

Tecnologias previstas:

* JDBC
* PostgreSQL ou MySQL

Conceitos praticados:

* Modelagem de dados
* SQL
* CRUD
* Persistência

### Fase 4 - Spring Boot

Transformação do projeto em uma API REST.

Objetivos:

* Cadastro de clientes
* Operações bancárias via API
* Integração com banco de dados

Tecnologias previstas:

* Spring Boot
* Spring Data JPA
* Hibernate
* Maven
* PostgreSQL

Conceitos praticados:

* APIs REST
* Injeção de Dependência
* Arquitetura em Camadas
* DTOs
* Boas práticas de desenvolvimento

## Status Atual

🚧 Em desenvolvimento

Atualmente o projeto encontra-se na fase inicial de modelagem das entidades e aplicação dos conceitos fundamentais de Programação Orientada a Objetos.
